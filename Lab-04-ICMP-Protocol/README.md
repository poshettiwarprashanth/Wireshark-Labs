# Part 1 - Understanding ICMP

## What is ICMP?

ICMP stands for:
**Internet Control Message Protocol**[cite: 1]

ICMP is a Layer 3 protocol used for:
- Network diagnostics[cite: 1]
- Error reporting[cite: 1]
- Connectivity testing[cite: 1]
- Troubleshooting network problems[cite: 1]

ICMP works at:
**OSI Model:** Layer 3 - Network Layer[cite: 1]

Unlike TCP and UDP, ICMP does not use port numbers[cite: 1].

### Examples of Port Usage in Other Protocols:
*   **TCP:**
    *   HTTP - Port 80[cite: 1]
    *   HTTPS - Port 443[cite: 1]
    *   SSH - Port 22[cite: 1]
*   **UDP:**
    *   DNS - Port 53[cite: 1]
    *   DHCP - Port 67/68[cite: 1]

ICMP works using:
- Type[cite: 1]
- Code[cite: 1]

---

# Important ICMP Fields

## Type
Type identifies the purpose of the ICMP message[cite: 1].

## Code
Code provides additional information about the message type[cite: 1].

> **Example:** 
> Type: 8, Code: 0 means an **ICMP Echo Request**[cite: 1].

|---|---|

For this lab, we focus mainly on:
- **Type 8** -> Echo Request[cite: 1]
- **Type 0** -> Echo Reply[cite: 1]

# Part 2 - How Ping Works

When we execute:
```bash
ping google.com
```[cite: 1]

Our computer sends an ICMP Echo Request packet[cite: 1].

### Communication Flow:

1. **Request Sent:**
   Your Laptop (192.168.0.6) $\rightarrow$ Google Server (142.x.x.x)[cite: 1]
   *   *ICMP Echo Request*[cite: 1]
   *   *Type: 8, Code: 0*[cite: 1]

2. **Reply Received:**
   Your Laptop (192.168.0.6) $\leftarrow$ Google Server (142.x.x.x)[cite: 1]
   *   *ICMP Echo Reply*[cite: 1]
   *   *Type: 0, Code: 0*[cite: 1]

The reply confirms that the destination host is reachable[cite: 1].

# Part 8 - Understanding Sequence Number

Sequence Number helps identify and match an ICMP request with its corresponding reply[cite: 1].

> **Example:**
> Request: *ICMP Echo Request, Sequence Number: 1*[cite: 1]
> Reply: *ICMP Echo Reply, Sequence Number: 1*[cite: 1]

The system understands the mapping:
- Request 1 $\longleftrightarrow$ Reply 1[cite: 1]
- Request 2 $\longleftrightarrow$ Reply 2[cite: 1]
- Request 3 $\longleftrightarrow$ Reply 3[cite: 1]

Sequence numbers help detect:
- Packet loss[cite: 1]
- Missing replies[cite: 1]
- Communication delays[cite: 1]

# Part 9 - TTL Analysis

TTL stands for **Time To Live**[cite: 1]. It tells how many routers a packet can pass through before being discarded[cite: 1]. Each router decreases the TTL value by 1[cite: 1].

### Hop Calculation Example:
*   Initial packet: $TTL = 128$[cite: 1]
*   Router 1 $\rightarrow$ TTL 127[cite: 1]
*   Router 2 $\rightarrow$ TTL 126[cite: 1]
*   Router 3 $\rightarrow$ TTL 125[cite: 1]

In our capture:
- **Request TTL** = 128[cite: 1]
- **Reply TTL** = 118[cite: 1]

**Approximate hops:** 
$$128 - 118 = 10\text{ hops}$$
[cite: 1]

# Part 11 - ICMP Security Perspective

Although ICMP is useful for troubleshooting, attackers can abuse ICMP traffic in several ways[cite: 1]:

### 1. ICMP Ping Sweep
Attackers use ICMP Echo Requests to discover active hosts for network reconnaissance and host discovery[cite: 1].
*Example:* `nmap -sn 192.168.1.0/24`[cite: 1]

### 2. ICMP Flood Attack
Attackers send a huge number of ICMP packets (also called a **Ping Flood Attack**) causing high CPU usage, network congestion, and service slowdowns[cite: 1].

### 3. ICMP Tunneling
Attackers hide information inside the ICMP payload[cite: 1]. Instead of normal ping data, they place commands, encrypted data, or sensitive information inside the packet[cite: 1].

### 4. ICMP Information Leakage
ICMP responses can inadvertently reveal information about the network structure, firewall behavior, or host availability[cite: 1].
